# Aegis icons

The shipping icons are **placeholders**, generated by `generate.py` from a
PIL-drawn master. They use the Aegis UI palette (pink → fuchsia gradient
on deep purple) so the desktop app feels coherent with the in-app theme.

Replace them with real artwork before public release — drop in your own
`parix-1024.png` and re-run the script, or hand-author each size.

## Files

| File | Used by |
|---|---|
| `parix.ico`     | Windows installer + window/tray icon |
| `parix.png`     | Linux `.desktop` entry + Electron tray fallback |
| `parix-*.png`   | Source PNGs at standard sizes (16, 32, 48, 64, 128, 256, 512, 1024) |
| `parix-1024.png` | Master render — also the input to the macOS `.icns` build |

## Regenerate

```bash
python aegis/electron/icons/generate.py
```

## macOS .icns

`.icns` requires Apple's `iconutil`, which only exists on macOS. Run this
on a Mac after `generate.py`:

```bash
cd aegis/electron/icons
mkdir parix.iconset
for size in 16 32 64 128 256 512 1024; do
  cp parix-$size.png "parix.iconset/icon_${size}x${size}.png"
done
iconutil -c icns parix.iconset -o parix.icns
rm -rf parix.iconset
```

The `electron-builder` config in `aegis/package.json` already points at
`electron/icons/parix.icns` for the Mac target — once the file exists the
packaged DMG picks it up.
